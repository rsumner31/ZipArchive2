ptsochantaris commented Feb 15, 2018

Hi @danwenzel and thanks for reporting this.

The first step in seeing what may be going wrong is logging Trailer's activity in the console, to see what or if some API call is failing, and why Trailer is not picking the problem up. You can activate it from Preferences -> Misc -> and tick "Log activity to console".

You can then track Trailer's sync activity in Console.app. Give it a try and try to see if some call is failing.

Additionally it may help to deactivate various repos, and see if one specific repo or subset of data is causing Trailer to exhibit the issue, to help test the issue faster.

Another thing to try is to try and disable various options (the most recent build had changes to querying for review stuff, so I would start by completely disabling those options to see if sync completes with them set to off)

If we can track down which call is breaking then it should be easy to see why that is. If you can provide a relatively short log from the sync activity where the issue occurs, I'll be more than happy to go through it and try to see what's "missing" from there.

Finally - when the sync hangs - can you see how many pending operations the app is reporting while the sync has "frozen"? You can see that from the top of the dropdown menu on the top-left of the main Trailer menu.

In any case, sorry you're having this issue - let's see if we can get to the bottom of this!

ptsochantaris commented Feb 15, 2018

Ah, that's very useful, thanks for posting it!

That 403 seems to be causing bad behaviour for some reason - the HTTP code is a "permission denied" result. Normally non-200 responses from the API for PRs and Issues will be 404 or 410, which indicate that item is gone.

So the possibilities are that (a) that item is gone but the GitHub API has a new way of expressing it, so I need to make Trailer aware of the new code, or (b) something is wrong with your permissions (i.e. you're not allowed access to that item) and Trailer should be failing the sync with a red "X"

In any case, Trailer should definitely not be "hanging" the sync indefinitely there - I just need to know which "direction" to fix it in (i.e. "item missing" or "permission was -correctly- denied")

To help determine this, can you see if you can view issue 272 in that repo from a browser and let me know what happens?
